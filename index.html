El sistema operativo Linux

  • Sistema tipo Unix, considerado uno de los ejemplos más destacados de software libre y código abierto.
  • Su núcleo ha sido desarrollado con los aportes de más de 15000 programadores.
  • Es utilizado en plataformas que van desde relojes inteligentes hasta supercomputadores.

    • El proyecto GNU

  • En 1983, Richard Stallman inició el proyecto GNU con el objetivo de crear un "sistema de software compatible con Unix y compuesto completamente de software libre".
  • GNU desarrolló editores, compiladores, líneas de comandos, interfaces gráficas y otras herramientas.

    • El kernel de Linux

  • En 1991, Linus Torvalds programó un kernel (núcleo) de sistema operativo que luego se denominó kernel de Linux.
  • En 1992, le asignó una licencia GPL de GNU, la cual permite su libre uso, modificación y distribución.

    • GNU/Linux

    El kernel de Linux y las herramientas de GNU fueron integrados en el sistema operativo Linux, también llamado GNU/Linux.

    Distribuciones ("distros") de Linux

  • Distribuciones de software que incluyen el kernel de Linux y una colección de aplicaciones seleccionadas para usos particulares (ej. científicos, empresariales, servidores).
  • Hay alrededor de 600 distros (cerca de 500 con desarrollo activo).

    • "Distros" populares

  • Debian (1993): muy estable y leal a los principios de software libre. Mantenida por voluntarios.
    • Ubuntu (2004): muy popular y enfocada en la usabilidad. Mantenida por Canonical Ltd.
    • Linux Mint (2006): hay una versión que desciende directamente de Debian y otra de Ubuntu. Mantenida por voluntarios.
    • Kali Linux (2013): orientada a análisis forense digital y pruebas de penetración. Mantenida por una organización de sombrero blanco.

    • "Distros" populares

  • Slackware (1993): la más antigua de las distribuciones activas. Muy simplista y Unix-like. Mantenida por su autor Patrick Volkerding.
    • SUSE Linux Enterprise Server (1994): es una de las primeras distros comerciales. Mantenida por la compañía SUSE.
    • openSUSE (2005): muy popular y amigable. Mantenida por la compañía SUSE y por voluntarios.

    • "Distros" populares

  • Red Hat Linux (1995): distribución comercial de la compañía Red Hat descontinuada en 2004.
    • Red Hat Enterprise Linux (RHEL) (2000): distribución comercial de Red Hat de código abierto y que vende soporte, entrenamiento y otros servicios.
    • Fedora (2003): otra distribución mantenida por Red Hat y también por el proyecto comunitario Fedora. Se usa para probar y madurar el código de RHEL.
    • CentOS (2004): distribución libre con licencia GPL compatible con el código fuente de RHEL. Mantenida por una comunidad de voluntarios.

    • "Distros" populares

  • Gentoo (2000): compila localmente el código fuente y lo optimiza para el tipo específico de computadora. Es para usuarios experimentados. Mantenida por la Fundación Gentoo.
  • Arch Linux (2002): distribución minimalista, para usuarios experimentados, que utiliza un modelo contínuo de liberación de software.
    • Manjaro Linux (2011): cuenta con interfaz gráfica y es más amigable para el usuario que Arch Linux.

    • Ubuntu

  • Está basada en la distribución testing de Debian, por lo que soporta versiones más actualizadas de software y más dispositivos de hardware.
  • Es una buena distribución para principiantes.

    • Ubuntu

  • Libera una nueva versión estándar cada 6 meses y una con soporte a largo plazo (LTS) cada 2 años.
  • Es una de las distros más populares y cuenta con documentación y soporte muy amplios en canales oficiales y no oficiales.

    • La interfaz de línea de comandos (CLI)

  • Permite ingresar comandos a través de líneas de texto.
  • Es más eficiente que las interfaces gráficas, pero menos intuitiva.
  • Los comandos de Linux tienen la sintaxis:
    comando opciones archivos
    Por ejemplo:
    ls -l *.txt

    • Generalidades de los comandos de Linux

  • Siempre se escriben en minúscula. IMPORTANTE: Linux diferencia las mayúsculas de las minúsculas.
  • Las opciones y los nombres de archivo no son obligatorios en los comandos, pero casi siempre se usan.
  • Las opciones siempre se escriben antes de los nombres de archivos.
  • El comando man (i.e. manual) brinda información de otro comando (ej. man ls).

    • Comandos: información general

    whoami: imprime el nombre del usuario. 
    
								$ whoami
								mfvargas
    hostname: imprime el nombre de la computadora (o "host"). 
    
								$ hostname
								eira
    uname: imprime información sobre el sistema. 
    
								$ uname -a
								Linux eira 4.19.0-1-amd64 #1 SMP Debian 4.19.12-1 (2018-12-22) x86_64 GNU/Linux
    pwd: imprime el directorio actual. 
    
								$ pwd
								/home/mfvargas

								$ touch archivo01.txt
								$ touch archivo02.txt
    Para añadir contenido a un archivo: 
    
								$ echo "Línea 01 Archivo 01" >> archivo01.txt
								$ echo "Línea 02 Archivo 01" >> archivo01.txt
								$ echo "Línea 01 Archivo 02" >> archivo02.txt

    Comandos: manejo de archivos

    more: imprime los contenidos de un archivo. 
    
								$ more archivo01.txt
								Línea 01 Archivo 01
								Línea 02 Archivo 01
    cat: concatena los contenidos de varios archivos. 
    
								$ cat archivo01.txt archivo02.txt > archivo03.txt
								$ more archivo03.txt
								Línea 01 Archivo 01
								Línea 02 Archivo 01
								Línea 01 Archivo 02
    wget: descarga un archivo de Internet. 
    
								$ wget http://www.textfiles.com/internet/usenet.txt

    Comandos: manejo de archivos

    cp: copia archivos. 
    
								$ cp archivo03.txt archivo04.txt
    rm: borra archivos. 
    
								$ rm archivo04.txt
    mv: mueve o renombra archivos. 
    
								$ mv archivo03.txt otronombre.txt

    Comandos: manejo de archivos

    mkdir: crea un directorio. 
    
								$ mkdir micarpeta
    cd: se cambia a un directorio. 
    
								$ cd micarpeta
    cambio al directorio raíz: 
    
								$ cd /
    cambio al directorio padre: 
    
								$ cd ..
    cambio al directorio "home" del usuario: 
    
								$ cd

    Comandos: otros

    clear: limpia la pantalla. 
    
								$ clear
    passwd: cambia la clave del usuario. 
    
								$ passwd

    Instalación de aplicaciones

    GDAL/OGR

    Se actualizan los repositorios y se instalan las aplicaciones: 
    
								$ sudo apt-get update
								$ sudo apt-get install gdal-bin libgdal-dev

    Instalación de aplicaciones

    R (https://www.digitalocean.com/community/tutorials/how-to-install-r-on-ubuntu-18-04)

    Se añade la llave y el repositorio: 
    
								$ sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys E298A3A825C0D65DFD57CBB651716619E084DAB9
								$ sudo add-apt-repository 'deb https://cloud.r-project.org/bin/linux/ubuntu bionic-cran35/'

    Se actualizan los repositorios y se instalan las aplicaciones: 
    
								$ sudo apt-get update
								$ sudo apt install r-base

    Ejecución de R e instalación de paquetes: 
    
								$ sudo -i R
								> install.packages("maptools")
								> install.packages("sp")
								> install.packages("randomForest")
								> install.packages("raster")
								> install.packages("rgdal")
								> install.packages("RSAGA")

    Instalación de aplicaciones

    QGIS (https://www.qgis.org/en/site/forusers/alldownloads.html#debian-ubuntu)

    Se agrega el repositorio a la lista de repositorios del sistema operativo: 
    
								$ sudo nano /etc/apt/sources.list
    Debe añadirse la línea: 
    
								deb https://qgis.org/ubuntu bionic main
    (En nano, se guarda el archivo con Ctrl-X)
    Se descargan las llaves, se actualizan los repositorios y se instalan las aplicaciones: 
    
								$ wget -O - https://qgis.org/downloads/qgis-2017.gpg.key | gpg --import
								$ gpg --fingerprint CAEB3DC3BDF7FB45
								$ gpg --export --armor CAEB3DC3BDF7FB45 | sudo apt-key add -
								$ sudo apt-get update
								$ sudo apt-get install qgis python-qgis qgis-plugin-grass
